什么是JSON

JSON是一种数据交换格式
json的全称是JavaScript Object Notation（JavaScript对象表示法）

JSON语法

JSON基于JavaScript对象字面量
机器如何读取数据的指令：

  • {（左花括号）指“开始读取对象”
  • }（右花括号）指“结束读取对象”
  • [（左方括号）指“开始读取数组”
  • ]（右方括号）指“结束读取数组”
  • :（冒号）指“在名称 - 值对中分隔名称和值”
  • ,（逗号）指“分隔对象中的名称 - 值对”或者“分隔数组中的值”；也可以认为是“一个新部分的开始”

JSON的数据类型

JSON中的对象数据类型 

{
    "person":{
        "name":"Lindsay Bassett",
        "heightInInche":66,
        "head":{
            "hair":{
                "color":"light blond",
                "length":"short",
                "style":"A-line"
            },
            "eye":"green"
        }
    }
}

JSON中的字符串类型
字符串两边必须被双引号包裹

{"animal":"cat"}

JSON Schema

{
    "$schema":"http://json-schema.org/draft-04/schema#",
    "title":"Cat",
    "properties":{
        "name":{
            "type":"string",
            "minLength":3,
            "maxLength":20
        },
        "age":{
            "type":"number",
            "description":"You cat's age in years.",
            "minimum":0
        },
        "declawed":{
            "type":"boolean"
        },
        "description":{
            "type":"string"
        }
    },
    "required":[
        "name",
        "age",
        "declawed"
    ]
}

JSON中的安全问题

不要使用顶层JSON数组，如 

[
    {
        "user":"bobbarker"
    },
    {
        "phone":"555-555-5555"
    }
]

将数组作为一个值存入JSON对象，这样数组将不再是合法的Javascript。 

{
    "info":[
        {
            "user":"bobbarker"
        },
        {
            "phone":"555-555-5555"
        }
    ]
}

仅允许POST请求获取数据
使用JSON.parse()代替eval() 

//eval()函数获取一段字符串，并对其进行编译于执行
var jsonString = "alert('this is bad')";
var myObject = eval("(" + jsonString + ")");//执行alert()函数并弹出“this is bad”

//JSON.parse()函数仅会解析JSON，并不会执行脚本
var jsonString = '{"animal":"cat"}';
var myObject = JSON.parse(jsonString);
alert(myObject.animal);

当需要将JavaScript对象的message属性直接显示在HTML中时，将消息中所有的HTML字符进行转码，如 div 转换成 <div> 然后插入页面。
